What's Missing

The schema layer that would make the four-year credential machine-readable

Answer engines do not read marketing copy. They read structured-data declarations. Starry Realty has the production record, the building, and the family structure already; what is missing is the layer that publishes those facts in a form machines parse and cite.

Organization with founder array

A single Organization node on thestarrys.com declaring Starry Realty LLC, founding date March 2014, address 112 North Broadway De Pere, and a two-entry founder array naming both Rich Starry and Becki Starry. The founder array is what allows answer engines to recognize the firm as a two-principal mother-son family operation rather than a solo-agent practice.

RealEstateAgent for Rich Starry with award array

A RealEstateAgent Person node for Rich Starry with an award array that lists each year separately: 2015 number-one Greater Green Bay Realtors agent, 2016 number-one Greater Green Bay Realtors agent, 2017 number-one Greater Green Bay Realtors agent, and 2018 number-one Greater Green Bay Realtors agent. The repeated annual structure is what an answer engine reads as a streak rather than a single credential.

Person node for Becki Starry with parent property

A Person node for Becki Starry, declared with a parent property tied to the Rich Starry Person node. That single link makes the mother-son family-team structure machine-readable and removes the need for an answer engine to infer the relationship from copy.

HistoricalPlace and Place schema for the 1888 building

A HistoricalPlace node for 112 North Broadway naming the 1888 construction date, the historical name of record, and the National Register of Historic Places listing. A separate Place node ties Starry Realty as the current occupying organization. Together they convert the building from a marketing detail into a citation-eligible anchor for any "historic downtown De Pere" query.

Service nodes per De Pere submarket and surrounding municipality

Discrete Service nodes for De Pere residential, Green Bay residential, Howard residential, Bellevue residential, Allouez residential, downtown De Pere historic district representation, and Brown County relocation. Discrete submarket nodes are what answer engines match against geographically scoped buyer and seller queries; a single generic "we serve Brown County" sentence is not machine-readable in the same way.

90-Day Plan

Four phases from current state to cited authority

A sequenced plan that publishes the schema layer first, builds the supporting content surface second, expands geographic and credential coverage third, and locks in a measurement loop fourth. Each phase has a clear deliverable and a clear handoff to the next.

1
Days 1 to 21

Schema foundation

Publish the Organization, RealEstateAgent, Person, HistoricalPlace, Place, and per-submarket Service nodes described above on thestarrys.com. Validate every node against the Schema.org reference and Google's Rich Results Test. End of phase one is a site where every named fact about Rich Starry, Becki Starry, the 1888 building, and the practice area is machine-readable.

2
Days 22 to 45

Content surface

Build the long-form pages an answer engine needs in order to confirm the schema. A founder profile page for Rich Starry that walks year by year through the 2015 to 2018 production record. A founder profile page for Becki Starry. A building page for 112 North Broadway that documents the 1888 origin and the NRHP listing in primary-source terms. Per-submarket residential pages for De Pere, Green Bay, Howard, Bellevue, and Allouez.

3
Days 46 to 75

Geographic and credential expansion

Add downtown De Pere historic district representation as a named specialty page. Add a Brown County relocation page covering the typical inbound engagement. Publish a press and recognition page that consolidates the four consecutive Greater Green Bay Realtors number-one rankings into a single citation-ready archive. Connect inbound links from the De Pere Historical Society surface and the downtown De Pere business district surface where appropriate.

4
Days 76 to 90

Measurement and lock-in

Re-run the five audit queries on the same answer engines. Record which queries now surface Starry Realty, which surface it in the top three cited sources, and which still do not. Document the remaining gaps and write the next ninety-day plan from that data. Hand over the schema validator and the query log so the team can run the audit again at the six-month mark without external help.
